## Задачи и чекпоинты (@tasks)

Каталог `tasks` используется для планирования и отслеживания крупных задач через детализированные чекпоинты.

### Принципы работы

- Перед началом работы над крупной задачей необходимо сформировать подробный список чекпоинтов, разбивая задачу на небольшие, измеримые шаги.
- Каждый чекпоинт должен быть сформулирован так, чтобы по нему было однозначно понятно, выполнен он или нет.
- План по чекпоинтам согласовывается при необходимости (с командой, заказчиком или мейнтейнером репозитория).

### Формат задач

- Для каждой крупной задачи создаётся отдельный файл в `tasks/` с понятным именем (например, `user-auth.md`, `ui-refactor-dashboard.md`).
- Внутри файла описываются:
  - краткое описание задачи;
  - контекст и цель;
  - список чекпоинтов с возможностью отмечать прогресс.

Пример структуры файла задачи:

```markdown
# Название задачи

## Цель
- Краткое описание, зачем выполняется задача.

## Контекст
- Важные детали, ограничения и договорённости.

## Чекпоинты
- [ ] Шаг 1: ...
- [ ] Шаг 2: ...
- [ ] Шаг 3: ...
```

### Отслеживание прогресса

- Каждый раз при завершении чекпоинта он должен быть сразу отмечен как выполненный в соответствующем файле задачи.
- Состояние чекпоинтов в файле должно всегда отражать фактический прогресс работы.

### Актуализация планов

- При изменении объёма или приоритетов крупной задачи необходимо своевременно обновлять список чекпоинтов:
  - добавлять новые;
  - уточнять формулировки существующих;
  - помечать как отменённые неактуальные пункты.
- Все изменения в планах должны фиксировать реальные договорённости и текущую стратегию выполнения задачи.
